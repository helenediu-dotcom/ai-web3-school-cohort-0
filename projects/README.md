# Projects

Week 4 项目构思和原型放这里。

## 目录约定

每个项目一个子目录，命名规则：`项目简称/`

```
projects/
└── 项目名/
    ├── design.md     # 项目设计文档（问题、方案、技术栈、MVP）
    ├── notes.md      # 开发笔记
    └── README.md     # 项目说明
```

## 可选方向（来自 Handbook）

- **Agentic Commerce**：Agent 如何发现服务、协商任务、完成支付
- **Wallet / Permission**：围绕钱包、权限、Session Key 做产品原型
- **AI Security**：攻击面、权限隔离、审计系统
- **Dev Tooling**：面向合约理解、测试、文档的工具
