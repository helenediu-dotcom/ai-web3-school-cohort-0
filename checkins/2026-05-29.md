## 核心收获

读完 Wallet / Permission Track Handbook，最重要的认知转变：**Agent 不应该"拥有"钱包，它只应该拥有一组可限制、可审计、可撤销的能力。** 钱包、智能账户或权限模块来持有资产和执行权，Agent 只提出意图、生成候选交易、请求有限权限，并在规则允许时触发动作。

七个核心知识点形成完整闭环：

1. **AI Wallet UX** — 用户看到 Agent 在做什么、涉及什么权限、有什么风险、怎么撤销
2. **Permission Policy** — 七维约束（资产范围、金额、合约、函数、价格/滑点、时间、频率）用代码写死，能被程序检查
3. **Session Key** — 临时受限 key，权限到期/额度用完/手动撤销后自动失效
4. **Safe Guard** — 确定性规则拦住硬违规，人工确认处理灰区
5. **ERC-4337 Workflow** — Agent 意图 → UserOp → Smart Account 校验 → Bundler → EntryPoint → 执行
6. **Pre-transaction Simulation** — 结构化模拟数据，不靠模型自然语言总结
7. **Recovery / Revocation** — 从设计阶段就内置撤销，不做事后补救
