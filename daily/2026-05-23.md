## 概念笔记

### Chain-aware Context

这次实践演示的关键链路：

```
RPC (eth_getTransactionByHash / eth_getTransactionReceipt)
  ↓ 返回原始 hex
Agent 解码 + 语义化（"input=0x 意味着纯转账"）
  ↓
塞进 LLM Context
  ↓
LLM 用自然语言解释 / 推理
```

**典型数据来源**：
- RPC 节点（`eth_call` / `eth_getBalance` / `eth_getLogs` / `eth_getTransactionByHash` …）
- 索引服务（Etherscan API、The Graph、Dune）
- 合约 Event 日志

**塞进 Context 的取舍**：
- **粒度**：整段 `logs` 可能上 KB，token 成本高 → 是否裁剪？
- **新鲜度**：RPC 实时拉 vs 用索引服务的缓存
- **结构化 vs 原始**：给 LLM 解码后的字段，还是让它自己解码 hex？前者省 token、降幻觉，但需要额外的工具层

## 实践记录

### 实践目标

把昨天那笔 Sepolia 自转交易的链上原始数据完整拉下来，用 AI 解码字段，理解「RPC 返回 → Agent Context → 自然语言解释」这条链路。

### 操作步骤

1. 选 RPC 节点：`https://ethereum-sepolia-rpc.publicnode.com`（公开免费、无需 API key）
2. 用 Node.js `https` 模块发 JSON-RPC POST：
   - `eth_getTransactionByHash` → 拿交易本身
   - `eth_getTransactionReceipt` → 拿执行结果（status / gasUsed / logs）
   - `eth_getBlockByNumber` → 拿区块时间戳
3. 把 hex 字段交给 AI 解码，对照 Etherscan 验证

### 关键字段解读

| 字段 | 原始 (hex) | 解码后 | 含义 |
|---|---|---|---|
| `chainId` | `0xaa36a7` | 11155111 | Sepolia 测试网 |
| `blockNumber` | `0xa64d16` | 10,898,198 | 区块高度 |
| `blockTimestamp` | `0x6a107ba0` | 2026-05-22 15:52:00 UTC | 出块时间 |
| `transactionIndex` | `0xa` | 10 | 该区块第 11 笔交易 |
| `nonce` | `0x0` | 0 | 这个钱包的第一笔交易 |
| `value` | `0x38d7ea4c68000` | 0.001 ETH | 1×10¹⁵ wei |
| `input` | `0x` | 空 | 纯转账，没调用合约 |
| `type` | `0x2` | EIP-1559 | 新版 Gas 模型 |
| `gas` (limit) | `0x7b0c` | 31,500 | 愿付上限 |
| `gasUsed` | `0x5208` | 21,000 | 实际消耗（协议硬编码） |
| `maxFeePerGas` | `0xb1aeed93` | ≈ 2.98 Gwei | 每 Gas 最高出价 |
| `maxPriorityFeePerGas` | `0x59682f00` | 1.5 Gwei | 给矿工的小费上限 |
| `effectiveGasPrice` | `0x97797277` | ≈ 2.5417 Gwei | 实际成交价 |
| 手续费 | — | 21000 × 2.5417 Gwei ≈ 0.0000534 ETH | 与 Etherscan 记录一致 ✅ |
| `status` | `0x1` | Success | 1 = 成功，0 = revert |
| `logs` | `[]` | 空 | 没触发 Event |

### 三个值得记住的语义信号

1. **`input: "0x"` + `logs: []` = 纯 ETH 转账**。一旦调用合约，`input` 会有 ABI 编码的字节码（前 4 字节是函数选择器），合约里的 `emit Event` 也会让 `logs` 不为空。这两个字段是判断「是不是合约交互」的关键。
2. **`gasUsed = 21000` 是协议硬编码**。任何普通 ETH 转账都恰好烧 21000。一旦调用合约会远超此值。所以 21000 本身就是个语义信号。
3. **EIP-1559 三档价格**：`maxFeePerGas` 是总价上限，`maxPriorityFeePerGas` 是给矿工的小费上限，`effectiveGasPrice` 是实际成交。倒推当时 baseFee ≈ 2.54 − 1.5 = 1.04 Gwei，是网络拥堵指标。

---

## 今日收获

链上原始数据全是 hex，本身不是 Agent 能用的 Context。真正有用的是「解码 + 语义化」之后的字段——例如 `input=0x` + `logs=[]` + `gasUsed=21000` 三者合起来就是一个强语义信号：「这是纯 ETH 转账，没有合约交互」。Chain-aware Context 的核心不是把链上数据塞给 LLM，而是先建一个解码/语义化层，再决定哪些字段进 Context。
